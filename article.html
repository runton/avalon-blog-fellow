Ubuntu16.10 下安装node.js提示./configure 权限不够的解决方法

流星 2011-10-25 HTML5/CSS 1255 34

最近在学习meteor，于是在自己的笔记本上装了个双系统Win7+Ubuntu16.10。第一次使用ubuntu系统，感觉很好。


接下来准备在Ubuntu上安装node。

1)  根据Node.js 安装提示文档 BUILDING.md 要求升级系统，并安装node.js依赖的包

       a)  sudo apt-get update

       b)  sudo apt-get install Python gcc make g++

 

2) 到node官网上下载   node-v6.9.4.tar.gz 源码，node源码压缩包我放在tmp文件夹下；

3) 在源码所在文件夹启动终端，输入命令：tar zxvf node-v6.9.4.tar.gz

     压缩包解压到当前目录：/tmp/node-v6.9.4

    

4) 现在进入源代码目录，然后运行./configure文件，如下图所示:

5) 界面提示“权限不够”；百度了一下，发现是文件夹下configure文件没有可执行的权限，所以要成功运行上面的命令，必须先设置configure为可执行文件。

使用chmod命令设置（需要管理员权限）：例如   chmod 777 文件名（目录文件名）；

chmod 命令在此不过多描述，想了解更多可以百度一下。


6）接下来在运行./configure ；运行结果如图


7）然后输入命令 make 进行编译，这个过程可能要花一段时间。请耐心等待。

8)   如果编译顺利完成，最后输入 ：make install 完成安装。


网络上还有很多方式安装node.js ；有一篇文章描述的很详细，大家有兴趣可以参考一下：Ubuntu 15.04下安装Node.JS的不同方式
